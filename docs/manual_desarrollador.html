Manual del Desarrollador

Arquitectura del Proyecto

El proyecto sigue una estructura modular para evitar la duplicación de código y facilitar el mantenimiento.

/
├── core/       # Configuración y clientes base
├── modules/    # Lógica de negocio por tipo de documento
├── run/        # Scripts de ejecución principal (Entry points)
├── tools/      # Utilidades de mantenimiento y migración
└── docs/       # Documentación técnica y de usuario

Núcleo (Core)

core/config.py: Centraliza la carga de variables de entorno, manejo de fechas y rutas base.

core/odoo_client.py: Encapsula la conexión XML-RPC con Odoo, manejando la autenticación.

🛠️ Herramientas Técnicas

Qué es Ubicación
Configuración de Rutas y Meses core/config.py
Conexión a Odoo core/odoo_client.py
Test de comunicación API tools/test_odoo.py

Módulos

Los módulos en modules/ contienen la lógica pesada que puede ser reutilizada:

  • documentos_module.py: Funciones para procesar adjuntos de account.move.
  • guias_module.py: Lógica dual de XML-RPC y automatización con Selenium.

Scripts de Ejecución

Ubicados en run/, son scripts ligeros que importan los módulos y ejecutan el flujo de trabajo.

Importante: Todos los scripts de ejecución añaden automáticamente la raíz del proyecto al sys.path para permitir importaciones absolutas.

Evolución del Flujo de Trabajo

Comparativa entre el flujo histórico (scripts de archive) y la nueva arquitectura modular.

Flujo Histórico (Manual)

  • Fragmentación: Scripts separados para Facturas, XML de Guías y Scraping de PDFs.
  • Concurrencia Manual: El operador lanzaba múltiples terminales para ganar tiempo.
  • Riesgo: Posibles inconsistencias entre el XML descargado y el PDF generado por Selenium.
  • Mantenimiento: Código duplicado en múltiples archivos .py.

Flujo Modular (Integrado)

  • Integridad: Un solo script asegura que cada XML tenga su PDF vinculado.
  • Concurrencia Segura: Uso de PIDs para carpetas temporales, permitiendo múltiples terminales sin conflictos.
  • Auto-Corrección: El script de Reparación escanea el disco real para encontrar faltantes.
  • Estandarización: Configuración centralizada en core/config.py.
