Migración de Rama: Master a Main

Introducción

Por estándares modernos de la comunidad de desarrollo, se recomienda utilizar main como el nombre de la rama principal en lugar de master. Esta guía detalla cómo realizar esta transición de forma segura.

1. Pasos en tu Computadora Local

Primero, debemos renombrar la rama en tu repositorio local:

# Renombrar la rama master a main
git branch -m master main
Dato: El parámetro -m significa "move" (mover/renombrar).

2. Pasos en el Repositorio Remoto (GitHub)

Ahora debemos subir la nueva rama y limpiar la antigua:

Subir la nueva rama main:

git push -u origin main

Eliminar la rama master en el remoto:

git push origin --delete master
Atención: Si master es la rama predeterminada en GitHub, el comando de borrado fallará. Debes ir a Settings > Branches en la web de GitHub y cambiar la "Default branch" a main antes de borrarla.

Resumen de Comandos

Copia y pega este bloque para una migración rápida:

git branch -m master main
git push -u origin main
git push origin --delete master
